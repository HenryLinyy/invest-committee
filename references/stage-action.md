# 階段 6：投資委員會結論——行動計畫規則

本階段是融合層：先 Bull/Bear 對辯，再依下列**確定性規則**產出行動計畫。全部公式可帶數字驗算。

## 前置：Bull case vs Bear case

各列 3 條最強論點，逐條必須引用前面階段的具體數據（不可空泛）。兩邊列完後才下結論——結論必須正面回應對方最強論點。

## 部位上限公式

```
建議部位上限 % = profile 單一部位上限 × 風險係數 × 風險承受度係數
（乘積不得超過 profile 單一部位上限本值）
```

| 風險雷達總分 | 風險係數 |
|---|---|
| 80-100 🟢 | 1.0 |
| 70-79 🟡 | 0.8 |
| 60-69 🟡 | 0.6 |
| 40-59 🔴 | 0.4 |
| <40 或一票否決 🔴 | 0（建議不建倉，僅追蹤） |

風險承受度係數（讀 `<資料目錄>/profile.md`，定位見 SKILL.md §2）：低 0.7／中 1.0／高 1.2。

驗算範例：上限 10%、風險分 72、承受度中 → 10% × 0.8 × 1.0 ＝ **8%**。

profile 有「總投資資金」時換算金額，並依市場最小交易單位取整：台股 1 張＝1000 股（不足一張時註明可用零股）；美股 1 股。「未設定」時只給 %。

## 進場區間

```
進場區間 = [最近強支撐位, min(現價, 支撐位 × 1.05)]
```

- 現價高於支撐 5% 以上 → 標「等回檔」，給觸發價（支撐位 × 1.05）。
- 分批規則：區間內分 2-3 批，首批 ≤ 建議部位的 50%。
- 支撐位取 stage-technical 小結的「最近強支撐」（強度標「強」或「中」者優先）。

## 停損位

```
停損位 = max(最近強支撐 × 0.97, 進場價 − 2 × ATR)
```

取兩者**較高者**（先到先停），同時輸出對應虧損 %（＝(進場價−停損位)/進場價）。ATR 取腳本輸出的 `atr`（絕對值，非 %）。

## 目標價

- 第一目標 ＝ 最近強壓力位（技術）。
- 第二目標 ＝ 機構目標價中位數（stage-consensus 有資料時）。
- 兩者並列且註明依據；無機構覆蓋只給第一目標。

## 重評觸發條件（報告固定輸出，逐條可驗證）

1. 收盤跌破停損位。
2. 台股：三大法人連 5 日賣超／美股：機構持股連兩季下降。
3. 月營收或財報低於共識（miss）。
4. 風險燈號轉 🔴。
5. 消息面重大利空（訴訟、處分、產業黑天鵝）。

任一觸發 → 建議重跑該股健檢（單環節風險模式即可）。

## 週期調整

| 週期（profile 預設或任務指定） | 支撐/乖離參考 | 估值權重 |
|---|---|---|
| 短線（<1個月） | bias_5/bias_10、5-10日支撐 | 低 |
| 波段（1-6個月） | bias_20/bias_60、20-60日支撐 | 中 |
| 長線（>6個月） | bias_120/bias_250、120-250日支撐 | 高（估值位階為主要進場依據） |

## 行動計畫表固定格式

| 項目 | 值 | 依據 |
|---|---|---|
| 結論 | 建倉/等回檔/觀望/不建倉 | 一句話 |
| 建議部位上限 | x%（≒金額/張數） | 公式帶入值 |
| 進場區間 | a ~ b | 支撐位依據 |
| 停損位 | c（虧損 x%） | max(支撐×0.97, 進場−2ATR) |
| 目標價 | 第一 d／第二 e | 壓力位／機構中位數 |
| 重評條件 | 上述 5 條 | — |

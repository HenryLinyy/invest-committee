# 報告模板（invest-committee 輸出）

通用規範：先結論後細節；缺資料寫「未獲取」不編造；表格數值精確到小數 2 位；每份報告尾附「資料來源與時效」清單與免責語「本報告為個人研究參考，最終投資決定自負。」

## 完整版（單股深度分析）

```
# {名稱}（{代碼}）投資委員會報告　{日期}

## ① 結論卡
一句話結論｜綜合評估：〔看多/中性偏多/中性/中性偏空/看空〕｜風險 xx 🟢🟡🔴｜建議部位上限 x%

## ② 公司快照
市場、產業、市值、現價（漲跌幅）、資料截至日

## ③ 基本面小結（stage-fundamental 三行格式）
## ④ 機構共識小結（stage-consensus 三行格式）
## ⑤ 技術與資金小結（stage-technical 三行格式）

## ⑥ 風險雷達
四維得分表＋燈號＋最重要風險 1-3 點

## ⑦ Bull vs Bear
多方 3 條（引用數據）／空方 3 條（引用數據）／結論如何回應對方最強論點

## ⑧ 行動計畫（stage-action 固定表格）

## ⑨ 資料來源與時效
逐項列來源（twstock MCP／富途／Yahoo／網搜網站名＋日期）與退回情況

## ⑩ 免責
本報告為個人研究參考，最終投資決定自負。
```

## 健檢精簡版（watchlist 批量）

先總表：

| 代碼 | 名稱 | 燈號 | 風險分 | 較上次變化 | 最重要變化一句話 |
|---|---|---|---|---|---|

（「較上次」比對 `<資料目錄>/reports/` 最近一份報告；無前次資料標「首次」。資料目錄定位見 SKILL.md §2。）

再逐檔 ≤5 行摘要：風險分變動原因、法人/資金動向、關鍵價位狀態。
持有部位（watchlist 有成本價）額外輸出：未實現損益 %＋狀態〔續抱／減碼觀察／觸發重評〕（依 stage-action 重評條件逐條檢查）。

## Dashboard 產出流程（完整版報告附帶）

1. 複製 `assets/dashboard-template.html` 到 scratchpad。
2. 替換全部 `{{TOKEN}}`（清單見模板頂部註解）。
3. 五軸雷達分數換算：基本面/共識/技術/資金 由各階段小結映射 0-100（判定「持續穩定成長」類最高檔=90、中檔=70、低檔=40、警戒=20），風險軸直接用風險總分。
4. `SendUserFile`（display: render）送出；不支援時存 `<資料目錄>/reports/` 並回傳路徑。

## 當沖版（當沖模式，限台股）

口徑見 [stage-daytrading.md](stage-daytrading.md)。輸出用 [assets/daytrading-dashboard.html](assets/daytrading-dashboard.html)（台股**紅漲綠跌**）。

- **單股當沖卡**：股頭（名/代碼/現價漲跌/類別 chip/可當沖 chip）→ 定性一行＋風險 badge＋持倉損益 → 五維雷達（軸標分數）→ 關鍵價位地圖（停損/支撐/現價/決戰帶/目標）→ 指標看板 kv（均線/乖離/RSI/KDJ/MACD/ATR/5日振幅/量比/成交值/法人/月營收 YoY/PE-PB）→ 當沖劇本（多空各一，含日內停損 ≤0.5×ATR）。填 `CARD_A`，`CARD_B`＝""、`COMPARE_SECTION`＝""。
- **雙股**：`CARD_A`＋`CARD_B` 兩張卡 → `COMPARE_SECTION`＝並排對照表＋委員會裁定（當沖優先序＋理由＋「做劇本不做信仰」）。
- **部位集中度警示**：留倉超 profile 單一部位上限 → 填 `WARN_HTML`（含超標倍數與 ATR% 資金晃動）；無則 `WARN_HTML`＝""。
- **能力邊界＋免責**填入 `FOOT`（日K-based、不含分時/五檔、要 tick 接富途、當沖尤甚）。
- 存 `<資料目錄>/reports/daytrading_<代碼>_<日期>.html`。

### 當沖 dashboard 產出流程
1. 複製 `assets/daytrading-dashboard.html` 到 scratchpad。
2. 每檔生成 `CARD_x` HTML（骨架見模板頂部註解；雷達 polygon 點與價位地圖 tick 座標算法同波段版）。
3. 替換全部 `{{TOKEN}}`，缺資料填「未獲取」，漲跌色用 class `up`(紅)/`down`(綠)。
4. 存檔／render。
